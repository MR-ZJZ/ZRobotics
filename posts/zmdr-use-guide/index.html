简介#

电机驱动是机器人技术中软件硬件桥梁。项目针对永磁同步电机（PMSM）伺服控制，从实现国产化替代，降低使用门槛，提高安全性等角度设计完整的伺服软硬件系统。其使用了新型电流环结构，具备参数自辨识、编码线性补偿、无感控制、MIT力矩、轨迹规划、队列响应等适用于多场合的控制方案，具备离轴编码、双编码、多圈记忆等位置操作功能。

USB用于与上位机通信，进行调试和校准。CAN通信为主要通信方式，具备最高的优先级，用于实际运动控制。另外提供UART接口，SPI接口，用于简易通信和编码器。

控制要点#

部署过程要点#

  • 无感与有感控制
    无感适用于无位置传感器的场合，具备速度波动小，噪声少等优势，但其无法在零速范围内保持稳定。有感则具备全速范围适应性，但是其存在编码器噪声、非线性安装误差等因素带来的控制精度影响。可以根据需求合理选择，可以同时开启有感和无感，驱动器会在合适时刻切换位置参考，发挥两者优势。
  • 编码非线性补偿
    对于编码器安装精度不高或使用不具备离轴非线性校准的编码器时，推荐打开线性补偿。但是对于齿槽转矩较大的电机，编码补偿可能受到齿槽转矩谐波的影响，导致精度下降。应根据情况合理选择。
  • CAN通信设置
    若总线存在其它协议设备时，注意配置合适的节点ID，避免协议冲突。最多支持14个节点，节点0xf为广播帧，节点0x0保留。可适当降低波特率来降低误码率。可设置CAN心跳反馈频率获取实时位置速度和电流。
  • 电机参数设置
    首先确定有无感控制模式、极对数和编码器,设置校准电流和最大电流，推荐校准电流为额定的一半。即可以进行参数校准。校准过程中可能产生各类错误，依据错误类型调整。如无错误，则系统不存在问题。可以进入使能控制。
  • 控制参数调节
    主要包含增益设置、滤波和容忍设置。增益调节的基本方法为，首先调整P，达到预期的响应性能，在适当增大I提高稳态性能，先调节速度环，再调节位置环。如电机有嗡嗡噪声，则需降低滤波系数(1表示不滤波)，过低的滤波系数可能导致震荡，应配合增益合理调节。在极端工况下，需要适当提高容忍系数以避免冲击导致的误报错。
  • 多圈位置记忆
    可以通过上位机上的“保存位置”按钮实现对当前位置的锁存，在系统断电重启后将直接从该位置启动。若断电后电机产生移动，在±0.4圈内驱动器自动补偿，超过0.4圈时，将产生“初位变动”警告。

运动控制要点#

  • 力矩控制
    力矩模式工作在MIT模式下，因此在扭矩模式给定设定电流的条件下不会持续旋转，若要持续加速，需要设置位置和速度增益为0。MIT模式有专用CAN帧格式。扭矩模式的最大速度受到最大转速(vli)参数的限制。
  • 点动控制
    对于速度和位置的简单点动(从一个目标到另一个目标)运动控制，请设置合适加减速，驱动将自动完成规划。过大的加减速将对系统产生冲击。可以通过配置响应模式(acm)实现到点反馈can消息。
  • 轨迹跟踪
    对于位置跟随要求较高的控制场合，有两种方式。其一是将加减速设置为0，同时上层控制器保持较高位置通信频率，实现简单跟随。对于频率受限场合，推荐使用PVT轨迹规划控制。在位置模式下，将队列间隔(pvt)设为控制帧间隔时间，即启动PVT模式，PVT模式有专用CAN帧格式。
  • 队列控制
    为降低系统对上层控制器实时性要求，驱动器提供队列缓冲，通过响应模式(acm)设置。轨迹执行时间间隔为队列间隔(pvt)的绝对值，对于速度模式而言，时间间隔的正负无影响。对于位置模式，时间间隔为正将执行轨迹跟踪，为负或0将执行点动。轨迹容量为6000点，超过容量将直接产生参数错误报错。可以通过队列控制(lif)实现启动和队列截取。例如
lif=0 lif=10000 lif=11000
 清空队列并停止执行   清空队列并开始执行   保留1000点并开始执行 

通信协议格式#

USB串口协议#

USB协议采用ASCII通信，格式为

1    读写指令 + UART CMD ID + 数据 + 结束符

读写段含读和写两种字符指令，分别为“r_”（read）和“w_”（write），数据段为对应写入参数的值，前加“=”，读指令无数据段；结束符为“\n”。 如写极对数为14，

1    w_pol=14\n

为了保证与上位机通信效率，写入不作反馈。可以通过读取确认参数写入是否正确。

1    r_pol\n

驱动器反馈

1    14.000000\n

所有数据应遵循参数表格所给出的参数范围，禁止写入的数据写入指令无效，参数超过范围或者指令格式不正确，指令无效。

CAN协议#

为充分利用CAN帧格式资源，采用类似CANOpen的CANID段拆分的方式进行通信。通信采用标准帧格式，CAN帧ID段如下描述

1|   ID b10~4   |   ID b3~0   |
2|  CAN CMD ID  | CAN NODE ID |

CAN帧DATA段即通信的数据，写入指令DATA段为4字节。如，驱动器节点ID为1，写极对数14，则CAN发送

1|   ID b10~4   |   ID b3~0   | DATA Byte0~3 |
2|      0xB1    |     0x1     |    14.0f     |

读取指令,将DATA段长度设置为0，如读取电机极对数，则CAN发送

1|   ID b10~4   |   ID b3~0   |
2|     0xB1     |     0x1     |

驱动器反馈

1|   ID b10~4   |   ID b3~0   | DATA Byte0~3 |
2|     0xB1     |     0x1     |    14.0f     |

若通信格式不合法或参数范围错误，指令无效，同时反馈一帧ERR。 CAN通信具备以下几类特殊格式：

  • 参数归一化
    为充分利用有限的数据空间，采用目标数据范围与无符号整型数据空间值一一对应的方式通信，称为归一。参数归一化范围如下
位置 速度 电流  位置增益   速度增益 
[pod,pou] [-vli,vli] [-ili,ili] [0,50] [0,50]

以位置为例，假设位置区间为[-10,100]，位置归一空间为两个字节。则当位置值小于等于-10时，归一位置为0，当位置值大于等于100时，归一位置为0xffff，位置为20时，对应的归一位置为

1    (20+10)/(100+10)*0xffff=17873

类似的，可以依据归一值计算真值，假设归一值为2000，则真值为

1    2000/0xffff*(100+10)-10=-6.643
  • MIT力矩模式帧格式(小端在右)
1| ID b10~4 | ID b3~0 | DATA b0~16 | DATA b16~27 | DATA b28~39 | DATA b40~51 | DATA b52~63 |
2|   0x38   | NODE ID | 归一目标位置 | 归一目标速度  | 归一目标电流  | 归一位置增益  | 归一速度增益  |

实际电流环目标电流表示为

1    位置P * (目标位置-实际位置) + 速度P * (目标速度-实际速度) + 目标电流
  • PVT轨迹跟踪帧格式(小端在左)
1| ID b10~4 |  ID b3~0  | DATA Byte0~4 | DATA Byte5~7 |
2|   0x39   |  NODE ID  |  归一目标速度  |  归一目标位置  |

轨迹控制含义为

1    在队列间隔(pvt)时间后，到达目标位置，到达时速度为目标速度
  • 心跳的内容(小端在左)
1| ID b10~4 |  ID b3~0  | DATA Byte0~3 | DATA Byte4~5 | DATA Byte6~7 |
2|   0x3A   |  NODE ID  |  归一实际位置  |  归一实际速度  |  归一实际电流  |

上位机#

上位机为用户调试电机提供更简单的交互方式，可以方便完成对驱动器的参数设定，校准，状态参数的监测和基本运动控制。上位机软件最大存储约30s的数据，通信频率1Khz。

分区功能#

  • 1区：设备选择和参数一键操作
    下拉选择驱动器设备，连接到错误设备将会自动断开。连接成功后会自动更新当前的模式和错误新信息。下方是对参数的一键操作，实现对用户参数的一键读取、写入、保存、擦除等。
  • 2区：用户参数区
    所有用户需要给定的参数，用户根据需求调节，部分参数是点击下拉窗口，可通过后方三角按钮单独发送该数据，也可通过一键写入发送全部数据，不发送则不会修改驱动器数据，发送后如果需要永久保存，需要点击保存参数以保存到FLASH，鼠标放在2区内滚动滚轮可以拉出下方隐藏的部分非常用参数。
  • 3区：模式和状态功能区
    可以设置当前模式，显示和操作当前的模式、错误和警告。“测试模式”用于检测硬件是否异常，非必要不进入。“参数校准”用于自动获取电机参数。“释放电机”将对电机失能。“刹车急停”将迅速锁定当前位置，其将造成冲击，非必要不使用。对于校准数据，功能区也提供了查询接口，也可写入当前位置和操作队列控制位。
  • 4区：绘图区
    包含三个通道，可以在上拉窗口选择对应示波参数，单击三角按钮开始示波，再次单击停止。点击暂停示波后，鼠标放在该区域内滚动滚轮缩放图像，以图像中间为中心缩放，按住滚轮拖动鼠标调整中心位置，总数据为窗口数据的三倍。
  • 5区：运动区
    可以拖动滑条设置目标参数，鼠标在本区内拖动才有效。也可直接在右侧输入框输入后单击按钮发送。滑条的最大、最小值可以用户更改，为了避免误操作，更改后回车才会有效。

基本操作流程#

  • 第一步：电机选择设备，选择正确设备后，点击后面连接图标，其变绿且出现状态信息时表示成功。选择了错误设备将出现“端口不匹配”的提示。

  • 第二步：点击“一键读取”，读取设备信息，更新上位机内用户参数。同样可以一键写入将上位机内的用户参数写入驱动器。

  • 第三步：根据控制对象设置用户参数的前8项内容，校准电流不宜过低，输出占空比低于3%将无法通过校准。点击3区的“参数校准”，电机开始运行自动完成校准程序。推荐在校准过程中打开电流、位置等监视信息。如校准过程产生错误，将在3区显示，若无错误，则表示校准完成。

  • 第四步：点击“选择模式”后面的下拉窗口选择运行的模式，点击后三角按钮实现模式的进入，当前模式将被更新。如在速度模式，则可打开速度监视，在5区中拖动滑条或直接输入目标数据，电机将开始运行。调节适当的控制器增益和滤波条件，达到需求效果。

  • 第五步：测试运行完成，点击“保存参数”完成对所有参数的锁存。驱动器自动重启，上位机断开。设备大概需要0.1s时间完成重启，重启完成后CAN会发送一帧自身节点ID消息。

Vofa兼容#

上位机浮点协议兼容Vofa，其具备更强大的数据存储功能，如果需要长时间高速数据监测，可采用vofa， 驱动器可同时以约1khz的速度打印四个数据。控制方式为

1    w_（通道）=（CAN CMD ID）\n

例如，通道a打印速度，

1    w_pwa=89\n

三个通道分别为pwa、pwb、pwc，pwd，89为0x59(vel)的十进制格式。关闭通道的方式为令通道值为0。

 80def motor_pvt():
 81
 82    delta_t = 0.002  # 控制时间间隔
 83    num = 3.14 / delta_t + 1  # 轨迹点数
 84    num_data = num
 85    tt = 0  # 轨迹时间统计参数
 86    can_write(0xf, mod_id, 11)  # 清除错误

 89    can_write(0xf, acm_id, 2)  # 设置为无反馈队列模式

 91    can_write(0xf, mod_id, 3)  # 进入位置模式

 94        if num_data > 0:
 95            # 在队列存储20个时启动队列，相当于给你20个缓冲点，队列边进边出

 16void can_pvt(uint8_t node_id, float v, float p, float vli, float pod, float pou) {
